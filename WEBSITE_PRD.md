## 6. Recommended Technology Stack

> [!IMPORTANT]
> **Recommended Stack: React + Vite + TailwindCSS**
> This stack is optimized for modern, interactive promotional sites with a rich ecosystem and excellent developer experience.

### 6.1 Primary Recommendation: React + Vite

| Layer | Technology | Rationale |
|-------|------------|-----------|
| **Framework** | [React](https://react.dev) | Industry standard, vast ecosystem, component-based |
| **Build Tool** | [Vite](https://vitejs.dev) | Extremely fast dev server and HMR |
| **Styling** | TailwindCSS v4 | Utility-first, fast development, consistent design |
| **Animation** | Framer Motion | Powerful, declarative animations for React |
| **Icons** | Lucide React | Clean, consistent SVG icons as components |
| **Routing** | React Router | Standard routing solution for SPAs |
| **Deployment** | Vercel / Netlify | Optimized for frontend frameworks |
| **Analytics** | Plausible or GA4 | Privacy-respecting options |

#### Why React + Vite?
- **Rich Interaction** â€“ Easier to implement complex animations and state management
- **Ecosystem** â€“ Access to massive library of components and tools
- **Performance** â€“ Vite ensures lightning fast development and optimized production builds
- **Maintainability** â€“ Component-based architecture scales well
- **Modern Standards** â€“ Uses latest ES modules and features

### 6.2 Alternative Options

| Stack | Best For | Trade-offs |
|-------|----------|------------|
| **Next.js (Static Export)** | If future blog/CMS needed | Heavier, more complex |
| **Vanilla HTML/CSS/JS** | Maximum simplicity | Manual optimization required |
| **Hugo/Eleventy** | Pure static sites | Less component-friendly |
| **SvelteKit** | If reactive features needed | Overkill for marketing site |

| **Build Command** | `npm run build` |
| **Publish Directory** | `dist` |
| **Node Version** | 18.x or 20.x |

### 12.2 Deployment Steps
1. Push code to GitHub repository
2. Connect repo to Netlify (free account)
3. Configure build settings
4. Enable automatic deploys on `main` branch

### 12.3 SEO Optimization for Subdomain Hosting

> [!IMPORTANT]
> Subdomain sites can be indexed by Google but may rank lower than custom domains. Follow these steps to maximize visibility.

**Essential SEO Actions:**
- [ ] Submit sitemap to Google Search Console (`hisava.netlify.app/sitemap.xml`)
- [ ] Add site to Bing Webmaster Tools
- [ ] Include JSON-LD structured data (MobileApplication, Organization, FAQPage)
- [ ] Link website from Play Store listing (most important backlink!)
- [ ] Add website link to all social media profiles
- [ ] Share on relevant communities (Reddit r/androidapps, Product Hunt, etc.)

**Netlify-Specific SEO Settings:**
```toml
# netlify.toml
[build]
  command = "npm run build"
  publish = "dist"

[[headers]]
  for = "/*"
  [headers.values]
    X-Frame-Options = "DENY"
    X-Content-Type-Options = "nosniff"
    Referrer-Policy = "strict-origin-when-cross-origin"
```

### 12.4 Future Migration to Custom Domain
When ready to upgrade:
1. Purchase domain (~$10-15/year from Namecheap, Porkbun, or Cloudflare)
2. Add domain in Netlify Dashboard â†’ Domain Settings
3. Update DNS records (Netlify provides instructions)
4. Enable HTTPS (automatic with Netlify)
5. Set up 301 redirects from old subdomain
6. Update Google Search Console with new domain
